🏘️ 小区PK · 当前已爬数据

当前默认口径是 `resale-only`。PK 仍分三套：一套看默认主源窗口的整体 CAGR；一套看精细到 `2b1b / 2b2b / 3b2b` 的真实户型 CAGR；一套看按面积近似分桶后的 `2b1b / 2b2b / 3b2b` CAGR，把三个 data bar 放在同一行里对比。当前统一规则是：`窗口不足12个月不显示 CAGR`。

📡 数据源与覆盖范围说明

当前前端数据源：`data/dashboard_data.json` / `data/dashboard_data.js`。这两个文件都由本地脚本从 `data/*_transactions.csv`、`data/appreciation_analysis.json`、`data/layout_mapping/*` 自动生成；若正式映射文件缺失，才回退到旧的 `data/poc_layout/*`。
基础口径（统一公式）：所有口径（1 / 2 / 3）共用同一 CAGR 公式——起始 PSF = 项目最早 `12个月` 交易的 `median PSF`；终止 PSF = 当月 `TTM`（过去12个月所有交易）的 `median PSF`；年化跨度 = `月份偏移 − 12`。至少需要 `24个月` 数据才能产出 CAGR。
口径 1：`主源整体 CAGR` 对当前每个项目默认 `resale-only` 主源窗口的全部成交应用基础口径公式；当前主源优先 `propertyforsale resale`，拿不到时用 `URA resale fallback`。
口径 2：`细户型 CAGR` 只展示当前已经做完正式 `2b1b / 2b2b / 3b2b` 映射的小区；每个户型在各自行内独立应用基础口径公式。
口径 3：`面积近似 CAGR` 用统一面积带粗分：`2b1b = 600-699 sqft`、`2b2b = 700-849 sqft`、`3b2b = 850-1199 sqft`；每个桶独立应用基础口径公式，并按三个桶里可用 `CAGR` 的平均值降序排。
注意：默认不再把 `New Sale / Sub Sale` 混进主 CSV。若某项目本地缓存里拿不到 resale，则该项目会退出默认集；历史 `SRX` 窗口只保留在 `data/srx/*` 作为备份，不再是默认主口径。

口径说明

当前数据口径

1. 全量小区表：直接来自本地 `data/*_transactions.csv`，默认只保留 `resale-only` 成交；当前优先用 `propertyforsale resale`，拿不到时用 `URA resale fallback`。
历史 `SRX` 窗口只继续保留在 `data/srx/*` 作为备份，表格来源列会同时列出主源和备份源。
2. 细户型分类：目前只对 Lakeville / Lake Grande 做了 `小户型 / 2b1b / 2b2b / 3b2b / 大户型` 映射，优先读取正式 `data/layout_mapping/*`，缺失时才回退旧 `poc_layout`。
3. 口径 1 PK：默认看当前 resale 主源窗口的整体 CAGR；先取年度中位 `PSF`，再按首末成交月份的实际月数精确年化，`propertyforsale resale` 主源和 `URA resale fallback` 主源都会进入。
4. 口径 2 PK：只看已经拿到 `2b1b / 2b2b / 3b2b` 数据的小区；每个桶都按首末成交月份精确年化，并分别展示对应窗口。
5. 口径 3 PK：按面积近似分桶，统一用 `600-699 / 700-849 / 850-1199 sqft` 粗分出 `2b1b / 2b2b / 3b2b`。
6. propertyforsale resale：这里指 propertyforsale 页面里显式标记为 `Resale` 的成交，不会默认混入 `New Sale / Sub Sale`。
7. URA fallback：这里只在本地拿不到 propertyforsale resale 时启用，并且只保留 `typeOfSale = 转售`。
8. 待确认：表示当前公开片段存在冲突，或者面积尚未拿到足够多的户型证据，不会强行归类。
